## 📚 Report Structure

### 🔹 Part A: Origin & Philosophy

* Why Git was created (Linux kernel crisis in 2005)
* Role of distributed version control
* GPL v2 license and its implications
* Ethics of open source and community contribution

### 🔹 Part B: Linux Footprint

* Installation via package managers (APT)
* Key directories and configuration files
* Update and security patch model
* User permissions and system integration

### 🔹 Part C: FOSS Ecosystem

* Dependencies: zlib, libcurl, OpenSSL, etc.
* Tools inspired by Git (GitHub, GitLab, CI/CD systems)
* Governance model (mailing list-based, no central authority)

### 🔹 Part D: Critical Analysis

* Git vs proprietary tools (GitHub, Perforce)
* Cost, security, flexibility comparison
* Final recommendation: Git as global standard

## 💻 Shell Scripts Included

This project includes **5 Bash scripts**, demonstrating practical Linux and scripting concepts:

### 1️⃣ System Identity Report

* Displays system info (kernel, distro, uptime)
* Shows Git version if installed

### 2️⃣ FOSS Package Inspector

* Checks if Git is installed
* Displays package metadata using `dpkg`
* Includes philosophical notes

### 3️⃣ Disk & Permission Auditor

* Audits directories like `/etc`, `/usr/bin`
* Displays permissions and disk usage
* Checks Git configuration file

### 4️⃣ Log File Analyzer

* Scans logs for keywords (default: "error")
* Counts occurrences and shows last matches
* Includes retry mechanism

### 5️⃣ Open Source Manifesto Generator

* Interactive script
* Generates a personalized OSS manifesto file
